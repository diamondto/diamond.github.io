1、关于技术

1、未来十年，一定是数字化时代，智能化时代。不好好学技术，可能要落后成文盲了，哈哈哈，风口来了🐷，愣着干啥？临渊羡鱼，不如退而结网。学如逆水行舟，不进则退。
2、所以馨兰慎重地选择了技术而不是产品，并不是头脑发热，而是思考之后的选择，选择了就不会后悔，既来之，则安之。选择没有对错，选择之后的最终结果，是主观心智与客观环境的调和，自己选择的路，啥姿态都会走完。
3、热爱是一方面，确实很重要，首长说的很对，兴趣是最好的老师。但是，馨兰想，兴趣不是一成不变的，敢于跳出舒适圈，做自己不擅长的事情，挖掘未知潜能，才是勇敢的人。只做擅长的、喜欢的、固化的事情，会失去创造力容易达到发展瓶颈，或者产生疲倦、迷茫。
4、学习能力是共通的，在某一方面做的很好，其他方面也不会很差。跨界发展，往往能破圈，看见未知的潜力。专攻固然重要，但容易陷入闭环，思维局限就只能闭门造车，闭门造车花费了时间和资源，最没有最大化产出效率，事倍功半。
5、职业发展的快乐应该有两种：A：做到自己曾经做不到的事情。B：做到别人以为你不可能做到的事情，把AB快乐当作动力和初心，KPI，绩效，晋级，就是水到渠成，马到成功的事情。荣誉是附属品、副产品；真正的追求应该是A和B。
6、前端发展这么广，横着挖，纵着挖，都有出路。它不香吗？风口来了，为蚂蚁金服卖命吧。😎🐷


前端脚手架BigFish

1、前端技术栈整合配置麻烦 🙃
2、开发联调麻烦🐷
3、前端资源发布上线麻烦👼

1、基于 umi 整合了前端工具链：包括Ant Design、Dva、Ant Design Pro 技术栈，最小化成本开发👍

2、简化开发联调：提供前端 http 接口的代理，在开发过程中随时切换 mock 和联调接口，方便自测和联调😎

3、一站式发布部署：从工程初始化，到研发迭代管理，到一键发布静态资源及页面渲染📌

1、tnpm run dev : 启动本地开发模式并启动应用，通过这种方式启动项目在修改文件之后不需要重复启动，除非是修改了model，config等配置文件。

2、tnpm run devs : 启动前后端联调模式，通过config.js中的配置找到后端的服务器，进行前后端的联调。

3、tnpm run test : 执行单元测试。

4、tnpx bigfish -v : 查看当前正在执行的bigfish的版本。

config.js配置

1、路由是一个数组，数组中每一个对象是一个路由信息，路由会从前到后匹配

2、可以在路由中使用属性：routes，继续用嵌套的方式匹配其他的路由

3、使用属性indexRoute，将当前的路径默认设置为展示某个component。或者在属性indexRoute中可以使用redirect重定向属性，指向某一个路径

4、使用history.push的方法在组件中跳转到指定路由

5、组件中可以通过props的属性params和location获取路由相关的参数

6、可以通过Link标签在html中进行页面跳转，外部链接使用a标签
